# 📘 Teoría de Git

## 🧠 ¿Qué es Git?

Sistema de control de versiones distribuido.

Permite:
- Versionar código
- Trabajar en equipo
- Volver a estados anteriores

---

## 📂 Estados

- Working Directory
- Staging Area
- Repository

---

## 🌿 Branching

Permite trabajar en paralelo sin afectar `main`.

---

## 🔀 Merge vs Rebase

### Merge
Une ramas con commit extra.

### Rebase
Reescribe historial.

---

## 🔁 Pull vs Fetch

- `fetch`: trae cambios
- `pull`: fetch + merge

---

## 🚀 Pull Requests

Permiten:
- Revisar código
- Aprobar cambios
- Integrar ramas

---

## ⏪ Reset vs Revert

- `reset`: modifica historial
- `revert`: crea commit inverso

---

## 🧭 Reflog

Historial completo de acciones (incluye commits borrados)

---

## 🏷️ Tags

Versionado (`v1.0`, etc)

---

## ⚠️ Conceptos clave

### HEAD
Puntero actual

### Conflictos
Cuando dos ramas modifican lo mismo

### Detached HEAD
Estado fuera de rama
